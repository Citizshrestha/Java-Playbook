Thymeleaf is a modern server-side Java template engine for web and standalone environments.

Unlike JSP, it focuses on natural templating that can be displayed on static prototypes and work correctly when processed.

Key Characteristics of Thymeleaf

  • Natural Templating: HTML files can be opened directly in browsers
  • Spring Boot Integration: Default template engine in Spring Boot
  • File Extension: .html

JSP Structure

A JSP page consists of several key components that work together to generate dynamic web content.

i) Directives

Provides instructions to the JSP web container about how to process the pages.

a) Page Directives: Defines page-dependent attributes like language, content-type, import statements, error pages, and session management. 
<%@ page language="java" contentType="text/html; charset=UTF-8" %>
b) Include Directives: Includes static files at translation time. 
<%@ include %>
c) Taglib Directive: Allows JSTL or custom tag libraries. 
<%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>

ii) Scriptlets

Allows Java code to be embedded within JSP pages during request processing.

  • Write Java code into HTML directly
<% 
    int a = 10;
    out.println(a);
%>

iii) Standard Actions

JSP standard actions are XML-based style tags that perform specific functions.

<jsp:include>: Includes resources dynamically at request time
<jsp:forward>: Forwards requests to other resources
<jsp:useBean>: Instantiates or locates JavaBeans
<jsp:param>: Passes parameters to other JSP actions
<jsp:getProperty>, <jsp:setProperty>: Manipulate Java properties
