Servlet Overview

A Servlet is a Java class that extends the capabilities of a server (typically a web server) to handle requests (especially HTTP). It generates dynamic content, such as interactive web pages or database interactions, forming the backbone of Java web applications.

i) Servlet Class Loading

  • When the first client request arrives, the web container loads the servlet class.
  • The container loads the .class file and uses the class loader to load it into memory.

ii) Servlet Instance Creation

  • After loading the servlet class, the web container creates a single instance (object) of the class.
  • Typically, only one instance is created per servlet to handle multiple requests efficiently.
  • The instance remains in memory throughout the servlet's lifetime.

iii) Initialization Phase - init()

  • The init() method is called immediately after instance creation.
  • It is invoked only once during the entire lifecycle.
  • It is used for one-time initialization tasks such as loading configuration parameters, establishing DB connections, or creating lookup tables.

iv) Request Handling - service()

  • The service() method is called for every client request.
  • It receives the HttpServletRequest and HttpServletResponse objects as parameters.
  • It processes the incoming request and handles the response generation.
  • It dispatches the request to methods like doGet(), doPost(), doPut(), and doDelete().

v) Destruction Phase - destroy()

  • This is called once when the container decides to remove the servlet from service (e.g., server shutdown).
  • It is used for closing files, releasing database connections, and performing other cleanup operations.
